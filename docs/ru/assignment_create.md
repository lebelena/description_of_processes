## № 1. Создание поручения

Создание поручения состоит из трёх событий: «Информация о поручении», «Подтверждение поручения» и «Загрузка документов».

К разным событиям доступ имеют разные контрагенты с определенными ролями в системе. В соответствии с бизнес-логикой по событиям «Информация о поручении» и «Загрузка документов» работает пользователь, который создал поручение, — автор. Доступ к «Подтверждению поручения» имеет только генеральный директор.

На данный момент реализована сквозная логика, когда пользователь выполняет работы по всем событиям в системе последовательно. BPMN-схема реализованного бизнес-процесса создания поручения находится на странице «BPMN-схема». Формы интерфейса с идентификаторами находятся на странице «Интерфейс». Последующее описание также предназначено для сквозной логики.

### 1.1. Точки входа в процесс

Создание поручения возможно из подсистемы «Справочники» → «Контрагенты», где необходимо выбрать конкретного контрагента и действующий договор, в рамках которого будет создано поручение. Последовательность шагов для основного и альтернативного сценариев описана в Таблицах 1.1–1.2.

Альтернативный сценарий при переходе к созданию поручения возможен, когда у контрагента по выбранному договору уже есть сформированные поручения. В этом случае в модальном окне появляется дополнительная вкладка «Поручения». Добавить новое поручение можно как на основной вкладке «О договоре», так и на вкладке «Поручения».
